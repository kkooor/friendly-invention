## 一、网络层概述

### 1. 核心任务

将分组从源主机沿网络路径传送到目的主机，实现**网络互连**和**数据包端到端传输**。

### 2. 在 TCP/IP 协议栈中的地位

在 TCP/IP 体系中，网络层被称为**网际层**，核心协议是 **IP**。

------

## 二、网络层提供的两种服务

| 比较维度     | 虚电路服务                | 数据报服务                        |
| ------------ | ------------------------- | --------------------------------- |
| **连接**     | 需要建立连接（虚电路 VC） | 无连接，每个分组独立转发          |
| **可靠保证** | 由**网络**保证可靠传输    | 由 ** 端系统（主机）** 保证可靠   |
| **地址携带** | 分组只需携带虚电路号      | 每个分组携带完整源 / 目的 IP 地址 |
| **典型实例** | ATM、帧中继、MPLS         | **因特网 (IP)**                   |

> 核心考点：因特网采用**无连接的数据报服务**，网络核心简单，复杂控制交由主机实现。

------

## 三、IP 地址（IPv4）

IPv4 地址总长 **32 位**，采用**点分十进制**记法，例：`192.168.1.1`。

### 1. 分类编址（早期方案，考试高频）

| 类别 | 高位前缀 | 首字节范围 | 默认子网掩码      | 用途        |
| ---- | -------- | ---------- | ----------------- | ----------- |
| A 类 | 0        | 1~126      | 255.0.0.0 /8      | 大型网络    |
| B 类 | 10       | 128~191    | 255.255.0.0 /16   | 中型网络    |
| C 类 | 110      | 192~223    | 255.255.255.0 /24 | 小型网络    |
| D 类 | 1110     | 224~239    | —                 | 组播 / 多播 |
| E 类 | 1111     | 240~255    | —                 | 科研保留    |

> 特殊 IP 地址（必背）
>
> 
>
> 1. `127.0.0.0/8`：环回地址，本地测试
> 2. `0.0.0.0`：本网络本主机
> 3. `255.255.255.255`：受限广播地址
> 4. 网络地址、广播地址不可分配给主机

### 2. 子网划分

- 原理：从**主机号高位借位**划分子网，合理划分网络。

- 子网掩码：网络位 + 子网位全 1，主机位全 0。

- 计算公式：

  

  可用主机数 

  =2m−2

  

  m

  = 剩余主机位数，

  减 2

  ：扣除网络地址、广播地址。

### 3. 无分类编址（CIDR）

- 格式：`IP/前缀长度`，例：`192.168.1.0/24`
- 特点：取消 A/B/C 类划分，地址按需分配，节约地址。
- 核心功能：**路由聚合（超网）**，提取多个网段最长公共前缀，简化路由表。

### 4. 地址规划技术

| 策略              | 说明                 | 优缺点                   |
| ----------------- | -------------------- | ------------------------ |
| FLSM 定长子网掩码 | 全网统一掩码         | 配置简单，地址浪费严重   |
| VLSM 变长子网掩码 | 不同子网使用不同掩码 | 地址利用率高，企业网主流 |

## 四、IP 数据报格式

### 首部整体

- 固定首部：**20 字节**，可选扩展首部，最大 60 字节。

| 字段         | 长度   | 核心作用                        |
| ------------ | ------ | ------------------------------- |
| 版本         | 4 bit  | IPv4=4，IPv6=6                  |
| 首部长度     | 4 bit  | 以 4 字节为单位，最小值 5       |
| 总长度       | 16 bit | 首部 + 数据，最大 65535 字节    |
| 标识         | 16 bit | 标识同一个 IP 数据报的所有分片  |
| 标志         | 3 bit  | DF 禁止分片、MF 还有分片        |
| 片偏移       | 13 bit | 分片位置，以 8 字节为单位       |
| TTL 生存时间 | 8 bit  | 每经过一台路由器 - 1，防止环路  |
| 协议         | 8 bit  | 上层协议：TCP=6、UDP=17、ICMP=1 |
| 首部检验和   | 16 bit | 仅校验 IP 首部，逐跳重新计算    |
| 源 / 目的 IP | 32bit  | 通信双方地址                    |

### 分片核心考点

1. MF=1：非最后一片；MF=0：最后一个分片
2. DF=1：禁止分片，MTU 不足直接丢弃报文
3. 片偏移单位：**8 字节**

## 五、IP 数据报的发送与转发

### 1. 主机转发规则

- 同网段：**直接交付**，ARP 解析目的 MAC 直接通信
- 不同网段：**间接交付**，转发给默认网关（路由器）

### 2. 路由器转发流程

1. 解封装二层帧，取出 IP 数据报
2. 根据**目的 IP**查询路由表
3. 转发原则：**最长前缀匹配**
4. 处理：TTL-1、重新计算首部检验和、重新封装二层帧
5. 无匹配路由：丢弃报文，返回 ICMP 目的不可达

## 六、路由协议

### 1. 路由分类

1. 配置方式：静态路由（手动）、动态路由（自动学习）

2. 作用范围：

   - IGP 内部网关协议：AS 自治系统内部（RIP、OSPF）
   - EGP 外部网关协议：AS 之间（BGP）

   

### 2. 三大核心路由协议对比（必考）

| 特性     | RIP                           | OSPF                     | BGP                 |
| -------- | ----------------------------- | ------------------------ | ------------------- |
| 协议类型 | IGP                           | IGP                      | EGP                 |
| 核心算法 | 距离矢量                      | 链路状态                 | 路径矢量            |
| 度量值   | 跳数（最大 15 跳，16 不可达） | 带宽代价 Cost            | 路由策略 / 路径属性 |
| 传输封装 | UDP 520 端口                  | 直接封装 IP（协议号 89） | TCP 179 端口        |
| 更新方式 | 周期全网更新                  | 触发式增量更新 LSA       | 仅传递最优路由      |
| 收敛速度 | 慢                            | 快                       | 较慢                |
| 适用场景 | 小型局域网                    | 中大型企业网 / 城域网    | 互联网骨干网        |

### 补充细节

1. **RIP**

   缺陷：坏消息传递慢（计数到无穷）

   优化：水平分割、毒性逆转、触发更新、抑制计时器

   

2. **OSPF**

   算法：Dijkstra 最短路径算法

   核心：划分区域 Area 0 骨干区域，减少 LSA 泛洪

   

3. **BGP**

   可靠传输：基于 TCP，保障跨网路由稳定

## 七、ICMP 因特网控制报文协议

- 定位：网络层辅助协议，封装在 IP 中，协议号**1**
- 核心作用：差错报告、网络诊断探测

### 两类报文

| 类型         | 常见报文                     | 应用           |
| ------------ | ---------------------------- | -------------- |
| 差错报告报文 | 终点不可达、时间超过、重定向 | 网络故障反馈   |
| 询问报文     | 回送请求 / 应答、时间戳      | 网络连通性测试 |

### 经典应用

1. Ping：ICMP 回送请求与应答，检测连通性
2. Traceroute：逐级递增 TTL，利用超时报文追踪路由

------

## 八、VPN 与 NAT

### 1. 私有 IP 地址（禁止公网路由）

- A 类：`10.0.0.0/8`
- B 类：`172.16.0.0/12`
- C 类：`192.168.0.0/16`

### 2. NAT 网络地址转换

- 作用：私有 IP ↔ 公网 IP 转换，多设备共享公网 IP
- 原理：端口映射，维护`内网IP:端口`与`公网IP:端口`映射表
- 优缺点：节约公网 IP、隐藏内网；破坏端到端通信

### 3. VPN

依托公网，通过**隧道 + 加密**技术，构建企业私有专用网络。

## 九、IPv6 协议

### 1. 设计背景

解决 IPv4 地址枯竭，地址长度升级为**128 位**。

### 2. 地址表示

- 格式：冒号十六进制
- 简化规则：前导零省略、连续全零段用`::`表示（仅 1 处）

### 3. 首部特点

固定**40 字节**，无首部检验和、无路由器分片，简化转发。

### 4. IPv4 & IPv6 核心对比

表格

| 特性     | IPv4           | IPv6               |
| -------- | -------------- | ------------------ |
| 地址长度 | 32 位          | 128 位             |
| 首部     | 20~60 字节可变 | 固定 40 字节       |
| 检验和   | 有             | 无                 |
| 分片     | 路由器可分片   | 仅源主机分片       |
| 地址配置 | 手动 / DHCP    | 支持无状态自动配置 |

### 5. 过渡技术

双栈技术、隧道技术、NAT64 地址转换。
