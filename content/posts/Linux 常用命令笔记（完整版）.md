## 一、which 命令

**作用**：查找系统中所使用的程序/命令的可执行文件具体路径（仅搜索系统 PATH 环境变量中配置的目录）。

**语法**：\`which 命令名\`

**注意**：仅能查找系统命令（可执行程序），无法查找普通文件。

**示例**：

```bash
which ls        # 查找 ls 命令的可执行文件路径，输出如 /bin/ls
which python3   # 查找 python3 程序的路径
which grep      # 查找 grep 命令的路径
```

## 二、find 命令

**作用**：在指定起始路径下，递归查找符合条件的文件/目录，支持按文件名、文件大小、类型、修改时间等多种条件筛选，功能强大。

### 1\. 按文件名查找（最常用）

**语法**：\`find 起始路径 \-name \&\#34;被查找文件名\&\#34;\`

**关键说明**：

- 起始路径：指定查找的根目录，\`/\` 表示全盘查找，\`\.\` 表示当前目录，\`\~/\` 表示用户家目录。

- 被查找文件名：支持通配符模糊查询，\`\*\` 匹配任意长度的字符（包括0个），\`?\` 匹配单个字符。

- 文件名区分大小写，若需忽略大小写，可使用 \`\-iname\` 替代 \`\-name\`。

**示例**：

```bash
find /home -name "test.txt"    # 在家目录（/home）下精准查找 test.txt 文件
find . -name "*.log"           # 在当前目录及子目录下，查找所有 .log 后缀的文件
find / -iname "test*"          # 全盘查找以 test 开头的文件，忽略大小写
find ~ -name "?.txt"           # 在用户家目录，查找文件名只有1个字符+ .txt 的文件（如 a.txt、1.txt）
```

### 2\. 按文件大小查找

**语法**：\`find 起始路径 \-size \+/\-n\[kMG\]\`

**关键说明**：

- \`\+\`：表示大于指定大小；\`\-\`：表示小于指定大小；无符号表示等于指定大小。

- n：表示大小数值（整数）。

- 单位：\`k\`（小写）= KB，\`M\` = MB，\`G\` = GB，不写单位默认是 512 字节（块）。

**示例**：

```bash
find /tmp -size +10M   # 查找 /tmp 目录下，大小大于 10MB 的文件
find . -size -5k       # 查找当前目录及子目录下，大小小于 5KB 的文件
find / -size 100M      # 查找全盘大小等于 100MB 的文件
find ~ -size +1G       # 查找用户家目录下，大小大于 1GB 的文件
```

## 三、grep 命令

**作用**：从指定文件中，通过关键字过滤文本内容，筛选出包含目标关键字的行，常用于日志分析、文本检索。

**语法**：\`grep \[\-n\] 关键字 文件路径\`

**选项说明**：

- \`\-n\`：可选选项，显示匹配内容的行号（便于定位）。

- \`\-i\`：可选选项，忽略关键字的大小写。

- \`\-v\`：可选选项，反向过滤，显示不包含关键字的行。

**注意**：若文件路径为 \`\-\`，表示从标准输入（如管道符传递的内容）中过滤。

**示例**：

```bash
grep "error" /var/log/messages  # 从系统日志中，筛选包含 error 的行
grep -n "root" /etc/passwd      # 查找 /etc/passwd 中包含 root 的行，并显示行号
grep -i "test" test.txt         # 查找 test.txt 中包含 test 的行，忽略大小写（匹配 Test、TEST 等）
grep -v "info" log.txt          # 查找 log.txt 中不包含 info 的所有行
```

## 四、wc 命令

**作用**：统计文件的字节数、字符数、行数、单词数，常用于文本统计。

**语法**：\`wc \[\-c \-m \-l \-w\] 文件路径\`

**选项说明**：

- \`\-c\`：统计文件的字节数（包含空格、换行符）。

- \`\-m\`：统计文件的字符数（UTF\-8 编码下，一个中文占1个字符）。

- \`\-l\`：统计文件的行数（以换行符 \`\\n\` 为计数标准）。

- \`\-w\`：统计文件的单词数（以空格、制表符、换行符为分隔符）。

**注意**：不指定任何选项时，wc 会默认输出 行数、单词数、字节数（顺序固定）。

**示例**：

```bash
wc -l test.txt      # 统计 test.txt 的行数
wc -w test.txt      # 统计 test.txt 的单词数
wc -c test.txt      # 统计 test.txt 的字节数
wc -m test.txt      # 统计 test.txt 的字符数
wc test.txt         # 默认输出：行数 单词数 字节数 test.txt
```

## 五、管道符 \|

**含义**：将管道符左边命令的执行结果，作为右边命令的输入（串联多个命令，实现复杂功能）。

**核心特点**：左边的输出 → 右边的输入，不生成中间文件，高效便捷。

**常用场景**：结合 find、grep、wc 等命令使用。

**示例**：

```bash
find . -name "*.txt" | grep "test"  # 先查找当前目录下所有 .txt 文件，再筛选文件名包含 test 的文件
ps aux | grep "nginx"               # 查看 nginx 进程（ps 查看所有进程，grep 筛选 nginx 相关）
cat test.txt | grep "error" | wc -l # 统计 test.txt 中包含 error 的行数
```

## 六、echo 命令

**作用**：在命令行中输出指定的文本内容，常用于测试、输出变量值。

**语法**：\`echo 输出的内容\`

**关键说明**：

- 输出内容若包含空格，需用双引号 \`\&\#34;\` 或单引号 \`\&\#39;\` 包裹。

- 支持转义字符，需加上 \`\-e\` 选项（如 \`\\n\` 换行、\`\\t\` 制表符）。

**示例**：

```bash
echo hello world          # 输出 hello world
echo "hello linux"        # 输出 hello linux（内容含空格，需用引号包裹）
echo -e "name\tage"       # 输出 name    age（\t 表示制表符，实现对齐）
echo -e "line1\nline2"    # 输出两行内容：line1 和 line2（\n 表示换行）
echo $PATH                # 输出系统 PATH 环境变量的值
```

## 七、反引号 \` \`

**作用**：将反引号内的命令执行，获取其输出结果，再将结果作为其他命令的参数使用（等价于 \`$\(\)\`，推荐使用 \`$\(\)\` 更易读）。

**语法**：\`命令1 \`命令2\`\` 或 \`命令1 $\(命令2\)\`

**示例**：

```bash
echo "当前目录下的 .txt 文件：`ls *.txt`"  # 先执行 ls *.txt，再将结果输出
cp `which ls` /tmp/                        # 查找 ls 命令路径，将其复制到 /tmp 目录
echo "当前日期：$(date)"                   # 执行 date 命令，输出当前日期（推荐用法）
```

## 八、重定向符 \&gt; 和 \&gt;\&gt;

**作用**：将左边命令的执行结果，写入到右边的文件中（替代默认的“输出到屏幕”）。

**区别**：

- \`\&gt;\`：覆盖写入 → 若目标文件已存在，会清空文件原有内容，再写入新内容。

- \`\&gt;\&gt;\`：追加写入 → 若目标文件已存在，会在文件末尾添加新内容，不破坏原有内容；若文件不存在，会自动创建文件。

**示例**：

```bash
echo "hello linux" > test.txt    # 将 hello linux 写入 test.txt，覆盖原有内容
echo "new line" >> test.txt     # 在 test.txt 末尾追加 new line
ls -l > file_list.txt            # 将 ls -l 的结果（当前目录文件详情）写入 file_list.txt
grep "error" log.txt >> error.log # 将 log.txt 中 error 相关行，追加到 error.log 中
```

## 九、tail 命令

**作用**：查看文件的末尾内容，常用于查看日志文件（实时监控日志更新）。

**语法**：\`tail \[\-f \-num\] 文件路径\`

**选项说明**：

- \`\-num\`：可选，指定查看文件末尾的 num 行（num 为整数），默认查看末尾10行。

- \`\-f\`：可选，实时监控文件变化，文件有新内容添加时，会自动刷新显示（需按 Ctrl\+C 退出监控）。

**示例**：

```bash
tail test.txt         # 查看 test.txt 末尾10行内容
tail -5 test.txt      # 查看 test.txt 末尾5行内容
tail -f /var/log/messages  # 实时监控系统日志，有新日志时自动显示
tail -100 /var/log/nginx/access.log # 查看 nginx 访问日志的最后100行
```

## 十、用户管理相关命令

**Linux 用户管理核心说明**：Linux 支持多用户、多用户组，一个用户可加入多个用户组；权限管控的单元分为用户级别和用户组级别，分别对应不同的权限范围。

### 1\. 组管理命令

**（1）groupadd 命令**

**作用**：创建新的用户组。

**语法**：\`groupadd 用户组名\`

**示例**：

```bash
groupadd testgroup  # 创建名为 testgroup 的用户组
```

**（2）groupdel 命令**

**作用**：删除指定的用户组（需确保该用户组无关联用户，否则删除失败）。

**语法**：\`groupdel 用户组名\`

**示例**：

```bash
groupdel testgroup  # 删除名为 testgroup 的用户组
```

### 2\. 用户管理命令

**（1）useradd 命令**

**作用**：创建新的系统用户。

**语法**：\`useradd \[\-g \-d\] 用户名\`

**选项说明**：

- \-g：指定用户所属的用户组（需先存在该用户组）。

- \-d：指定用户的家目录（默认家目录为 /home/用户名）。

**注意**：不指定 \`\-g\` 选项时，系统会自动创建一个与用户名同名的用户组，并将该用户加入该组。

**示例**：

```bash
useradd testuser          # 创建 testuser 用户，自动创建同名用户组
useradd -g testgroup testuser  # 创建 testuser 用户，并指定其所属用户组为 testgroup
useradd -d /home/myuser testuser  # 创建 testuser 用户，指定家目录为 /home/myuser
```

**（2）userdel 命令**

**作用**：删除指定的系统用户。

**语法**：\`userdel 用户名\`（仅删除用户，保留家目录）；\`userdel \-r 用户名\`（删除用户同时删除其家目录）。

**示例**：

```bash
userdel testuser      # 删除 testuser 用户，保留家目录
userdel -r testuser   # 删除 testuser 用户，同时删除其家目录
```

**（3）getent passwd 命令**

**作用**：查看系统中所有用户的详细信息（包括用户名、用户ID、用户组ID、家目录、登录shell等）。

**语法**：\`getent passwd\`

**示例**：

```bash
getent passwd  # 查看系统全部用户信息
```

## 十一、权限管理相关命令

**权限基础说明**：Linux 中文件/文件夹的权限分为读、写、执行三种，对应符号和数字如下：

- r：读权限（读取文件/查看文件夹内容），对应数字 $4$。

- w：写权限（修改文件/创建/删除文件夹内文件），对应数字 $2$。

- x：执行权限（运行可执行文件/进入文件夹），对应数字$1$。

权限组合可通过数字求和表示（如 rw\- 对应 $4+2=6$，rwx 对应 $4+2+1=7$）。

### 1\. chmod 命令

**作用**：修改文件或文件夹的权限信息。

**注意**：只有该文件/文件夹的所属用户或 root 用户（超级用户）有权限修改。

**语法**：\`chmod \[\-R\] 权限 文件或文件夹\`

**选项说明**：

- \-R：递归修改，对文件夹内的所有文件和子文件夹应用相同的权限设置（仅用于文件夹）。

**示例**：

```bash
chmod 755 test.txt    # 给 test.txt 文件设置权限：所属用户 rwx（7），组用户 r-x（5），其他用户 r-x（5）
chmod 644 test.txt    # 给 test.txt 文件设置权限：所属用户 rw-（6），组用户 r--（4），其他用户 r--（4）
chmod -R 777 testdir  # 递归设置 testdir 文件夹及内部所有内容的权限为 rwxrwxrwx（所有用户拥有全部权限）
```

### 2\. chown 命令

**作用**：修改文件或文件夹的所属用户和所属用户组。

**注意**：仅 root 用户或文件/文件夹的所属用户可执行该操作。

**语法**：\`chown \[\-R\] \[用户\]:\[用户组\] 文件或文件夹\`

**选项说明**：

- \-R：递归修改，对文件夹内的所有文件和子文件夹应用相同的所属用户/组设置（仅用于文件夹）。

- \`:\`：用于分隔所属用户和所属用户组（可省略用户组，仅修改所属用户；也可省略用户，仅修改所属用户组，格式为 \`:用户组\`）。

**示例**：

```bash
chown testuser:testgroup test.txt  # 将 test.txt 的所属用户改为 testuser，所属用户组改为 testgroup
chown testuser test.txt            # 仅将 test.txt 的所属用户改为 testuser
chown :testgroup test.txt          # 仅将 test.txt 的所属用户组改为 testgroup
chown -R testuser:testgroup testdir # 递归修改 testdir 文件夹及内部所有内容的所属用户和用户组
```

## 补充说明

- 所有命令的语法中，\`\[\]\` 内的内容为可选参数，根据需求选择是否添加。

- 命令区分大小写，如 \`ls\` 和 \`LS\` 是两个不同的命令（Linux 中默认大小写敏感）。

- 若命令执行失败，大概率是路径错误、权限不足（可尝试加 \`sudo\` 提升权限）或语法错误。
