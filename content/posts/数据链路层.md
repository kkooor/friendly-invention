## 一、数据链路层的定位与功能

### 1. 在OSI/RM中的位置
位于物理层之上、网络层之下，利用物理层提供的比特传输服务，为网络层提供**帧为单位**的、**基本可靠**的数据传输。

### 2. 核心功能
- **封装成帧（Framing）**
- **透明传输（Transparent Transmission）**
- **差错检测（Error Detection）**
- **流量控制（Flow Control）**
- **差错控制 / 可靠传输（Error Control）**
- **介质访问控制（MAC，在广播信道中）**

### 3. 数据单元
数据链路层协议数据单元为 **帧（Frame）**。

---

## 二、三个基本问题及解决方法

| 基本问题     | 目标                                             | 典型解决方案                                                 |
| :----------- | :----------------------------------------------- | :----------------------------------------------------------- |
| **封装成帧** | 确定帧的边界，将比特流划分为帧                   | 字符计数法、字符填充法（PPP异步）、零比特填充法（HDLC）、物理编码违例法（以太网前导码） |
| **透明传输** | 帧内任何比特组合均可透明传送，不会被误判为帧边界 | 字节填充（转义字符 `0x7D`）、零比特填充（每5个连续1后插入0） |
| **差错检测** | 检测传输过程中出现的比特差错                     | 奇偶校验、循环冗余检验（CRC）、校验和（CheckSum）            |

## 三、成帧方法详解

### 1. 字符计数法
在帧头设一字段指明帧中字符数。  
**缺陷**：计数字段出错会导致后续所有帧边界错乱。

### 2. 字符填充法
- **定界符**：ASCII字符 `SOH`（`01H`）和 `EOT`（`04H`），或 PPP 中的 `0x7E`。
- **转义机制**：出现定界符或转义字符时，前插转义字符（`ESC`，`1BH` 或 `0x7D`），并将原字符变换后传送。

### 3. 零比特填充法（HDLC 风格）
- **帧定界符**：`01111110`（`0x7E`）。
- **填充规则**：发送时在数据中每遇到 **5 个连续 1** 后自动插入一个 `0`；接收时自动删除。

### 4. 物理编码违例法
利用物理层编码中的无效信号模式标识帧开始/结束（如以太网采用的前导码+帧首定界符，以及无效曼彻斯特编码信号）。

## 四、差错检测技术

### 1. 奇偶校验
简单但检错能力弱，只能检出奇数个比特错误。

### 2. 循环冗余检验（CRC）
- **原理**：将比特串看作多项式系数，模 2 除法求余数。
- **步骤**：
  1. 待发送数据 $M$，生成多项式 $G$（$r+1$ 位）。
  2. $M$ 后补 $r$ 个 $0$，模 2 除以 $G$，得余数 $R$（$r$ 位）。
  3. 发送序列 $= M + R$。
  4. 接收端用相同 $G$ 除接收序列，余数为 $0$ 则无差错。
- **常用生成多项式**：
  - **CRC-16**：$x^{16} + x^{15} + x^2 + 1$
  - **CRC-CCITT**：$x^{16} + x^{12} + x^5 + 1$
  - **CRC-32**（以太网使用）：  
    $x^{32} + x^{26} + x^{23} + x^{22} + x^{16} + x^{12} + x^{11} + x^{10} + x^8 + x^7 + x^5 + x^4 + x^2 + x + 1$

### 3. 校验和（CheckSum）
常用于高层协议（IP、TCP、UDP），数据链路层 PPP 也采用。

## 五、流量控制与可靠传输协议

### 1. 停止-等待协议（Stop-and-Wait）
- 每发一帧即停止，等待确认后再发下一帧。
- **优点**：简单。
- **缺点**：信道利用率低。
- 需要**超时重传**机制应对帧或确认丢失。

### 2. 连续 ARQ 协议（滑动窗口协议）

#### (1) 后退 N 帧 ARQ（Go-Back-N）
- 发送窗口 $> 1$，接收窗口 $= 1$。
- 出错帧及之后的所有帧均需重传。

#### (2) 选择重传 ARQ（Selective Repeat）
- 发送窗口 $> 1$，接收窗口 $> 1$。
- 只重传出错帧，接收方缓存正确但未按序到达的帧。

#### (3) 信道利用率
$$
U = \frac{ \text{发送窗口大小} \times T_f }{ T_f + 2\tau + T_{ack} }
$$
其中：
- $T_f$：发送一帧的时间
- $\tau$：单向传播时延
- $T_{ack}$：发送确认帧的时间

## 六、介质访问控制（MAC）子层

数据链路层在广播信道中划分为 **LLC（逻辑链路控制）** 和 **MAC（介质访问控制）** 两个子层。

### 1. 信道划分协议（静态划分）
- 频分复用（FDM）
- 时分复用（TDM）
- 波分复用（WDM）
- 码分复用（CDM）

### 2. 随机访问协议

#### (1) ALOHA
- 纯 ALOHA
- 时隙 ALOHA

#### (2) CSMA（载波监听多点接入）
- 1-坚持 CSMA
- 非坚持 CSMA
- p-坚持 CSMA

#### (3) CSMA/CD（载波监听多点接入/碰撞检测）
- **应用场景**：**有线以太网**。
- **核心机制**：
  - 先听后说
  - 边听边说
  - 冲突停说
  - 随机重传
- **关键计算**：
  - 冲突检测条件：**往返传播时延（争用期）** 内可能发生冲突。
  - **最小帧长** $=$ 争用期 $\times$ 数据速率。
- **退避算法**：**二进制指数退避**（重传次数 $k$，退避时间 $=$ 随机整数 $\times$ 争用期）。

#### (4) CSMA/CA（碰撞避免）
- **应用场景**：**无线局域网（Wi-Fi）**。
- **原因**：无线环境中碰撞检测困难（隐蔽站、暴露站问题）。
- **机制**：
  - 帧间间隔（IFS）
  - 随机退避
  - RTS/CTS 预约信道

### 3. 受控访问协议
- 轮询（Polling）
- 令牌传递（Token Passing）：令牌环网、FDDI

## 七、以太网（Ethernet）

### 1. IEEE 802.3 标准
| 类型       | 速率     | 常见标准                       |
| :--------- | :------- | :----------------------------- |
| 传统以太网 | 10 Mbps  | 10BASE5、10BASE2、10BASE-T     |
| 快速以太网 | 100 Mbps | 100BASE-TX、100BASE-FX         |
| 千兆以太网 | 1 Gbps   | 1000BASE-T、1000BASE-SX/LX     |
| 万兆以太网 | 10 Gbps  | 仅支持全双工，不再使用 CSMA/CD |

### 2. MAC 帧格式（以太网 V2）

| 字段       | 长度           | 说明                   |
| :--------- | :------------- | :--------------------- |
| 前导码     | 7 字节         | 同步用                 |
| 帧首定界符 | 1 字节         | 标识帧开始             |
| 目的地址   | 6 字节         | 目标 MAC 地址          |
| 源地址     | 6 字节         | 源 MAC 地址            |
| 类型/长度  | 2 字节         | 上层协议类型或数据长度 |
| 数据       | 46 ~ 1500 字节 | 有效载荷               |
| FCS        | 4 字节         | CRC-32 校验            |

- **最小帧长**：64 字节（数据字段至少 46 字节）。
- **最大帧长**：1518 字节（不计前导码和 VLAN 标签时）。

### 3. 以太网交换机与网桥
- **工作原理**：自学习源 MAC 地址构建 **MAC 地址表**，按目的 MAC 地址转发（若未知则泛洪）。
- **冲突域隔离**：交换机每个端口是一个独立冲突域。
- **广播域**：默认所有端口处于同一广播域（可通过 VLAN 划分）。
- **生成树协议（STP）**：防止环路，标准为 IEEE 802.1D。

## 八、虚拟局域网（VLAN）

### 1. 目的
在不改变物理拓扑的情况下，将局域网逻辑划分为多个广播域，提高安全性和灵活性。

### 2. IEEE 802.1Q 标签
在以太网帧中插入 4 字节标签（TPID + TCI），其中 **VLAN ID** 占 12 位，可标识 4094 个 VLAN。

### 3. 端口类型
| 类型        | 连接对象      | 帧标签                          |
| :---------- | :------------ | :------------------------------ |
| Access 端口 | 终端设备      | 出入帧均不带标签                |
| Trunk 端口  | 交换机/路由器 | 承载多个 VLAN，帧带 802.1Q 标签 |

## 九、点对点协议（PPP）

### 1. 应用场景
用户计算机到 ISP 的拨号连接、路由器间串行链路。

### 2. 组成部分
- **封装成帧**：使用 `0x7E` 定界，异步链路采用字符填充。
- **链路控制协议（LCP）**：建立、配置、测试链路。
- **网络控制协议（NCP）**：为不同网络层协议（如 IP）协商参数。

### 3. 工作状态链路静止 → 链路建立（LCP） → 鉴别（可选） → 网络层协议配置（NCP） → 链路打开 → 链路终止

- **HDLC（高级数据链路控制）**：面向比特的同步协议，典型帧格式包含标志、地址、控制、信息、FCS 字段。
